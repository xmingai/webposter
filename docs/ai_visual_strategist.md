# WebPoster：从 AI 作图工具到 AI 视觉策略师

> 产品哲学与技术路线文档
> 撰写日期：2026-04-01
> 基于：美图设计室逆向分析 · Google Stitch 架构研究 · 三模型对比实验

---

## 一、核心命题

**当人人都能用 AI 生成图片时，什么变得稀缺？**

```
传统时代                          AI 时代
─────────                        ─────────
稀缺 = 执行力                    稀缺 = 创意方向 + 审美判断 + 业务洞察
（会 PS、会画画、会拍照）          （知道该做什么图、为什么做、怎么验证效果）

廉价 = 想法                      廉价 = 执行力
（谁都有想法）                    （人人都能生成图片）
```

**WebPoster 的产品定位**：

> WebPoster 不是一个「AI 作图工具」，而是一个「AI 视觉策略师」。
>
> 它通过 editableActions 提供创意方向，通过数据闭环验证创意效果，
> 用"洞察 → 提案 → 执行 → 度量 → 迭代"的飞轮，
> 持续为用户提供**比用户自己更懂市场的视觉策略**。

## 二、AI 设计师的「职级跃迁」

这个定位映射了真实世界中设计师的职业成长路径：

| 职级 | 核心能力 | 对应的 AI 产品水平 | 代表产品 |
|------|---------|------------------|---------|
| **初级设计师** | 能做图 | 文生图工具 | Midjourney、SD WebUI |
| **高级设计师** | 做好图 | 模板化设计 + AI 美化 | 美图设计室、Canva |
| **设计总监** | 用视觉驱动业务 | 🎯 **WebPoster（目标）** | 尚无成熟产品 |

### 三个层级的本质区别

```
初级：用户说"做一张海报" → AI 做一张海报
高级：用户说"做一张好看的海报" → AI 做一张好看的海报
总监：用户说"帮我提升产品点击率" → AI 输出一套视觉策略
      包括：做什么图、为什么做、做完怎么验证、验证后怎么迭代
```

**市面上 99% 的 AI 设计工具停在了"高级设计师"层级**——图很漂亮，但不知道为什么做这张图、这张图要达到什么商业目标。

从"高级"到"总监"的跃迁关键，在于一个字——**"懂"**：

| | 高级设计师（当前 AI 工具） | 设计总监（WebPoster 目标） |
|---|---|---|
| **输入** | "帮我做一张海报" | "帮我提升这个品的点击率" |
| **思考** | 配色好不好看 | 这个品类的用户更偏好什么视觉风格 |
| **输出** | 一张精美的图 | 一套带 A/B 测试方案的视觉策略 |
| **反馈** | 无 | 根据数据自动迭代下一版 |

## 三、核心引擎：editableActions

### 3.1 为什么 editableActions 是产品的灵魂

`editableActions` 不是"功能菜单"，而是**"AI 创意总监的提案簿"**。

它的价值不在于"帮用户做一件他已经想好的事"，而在于——

> **告诉用户他应该做什么、为什么做。**

当用户上传一张图片，传统工具等着用户下指令；而 WebPoster 主动输出提案：

```
┌──────────────────────────────────────────────┐
│  🧠 AI 视觉策略师分析结果                       │
│                                              │
│  检测到：这是一张外卖平台的菜品主图               │
│  当前问题：背景杂乱，蛋黄色泽偏暗                 │
│                                              │
│  📌 推荐修改方案（按预估转化率排序）：              │
│                                              │
│  1. ⭐ 增强蛋黄流心效果                         │
│     预估影响：食欲感 +40%，同品类 CTR Top 10%    │
│                                              │
│  2. 🎨 换为原木纹理背景                         │
│     预估影响：高级感显著提升，适合客单价 >50 元    │
│                                              │
│  3. 🧹 移除背景杂乱餐具                         │
│     预估影响：视觉焦点集中度 +60%                │
│                                              │
│  一键全部应用 ✨  |  逐项预览                    │
└──────────────────────────────────────────────┘
```

### 3.2 editableActions 的三层意图模型

每张图片的 editableActions 应覆盖三个层次的意图：

#### Layer 1：物理层 —— "修好它"
- 去水印、修复模糊、移除路人、扩图补边
- **用户心理**：这是基础需求，AI 应自动检测并默认展示
- **Alpha Moment**："它连水印都自动帮我去了？"

#### Layer 2：商业层 —— "卖掉它"
- 换背景（居家→工作室）、添加促销角标、适配平台尺寸、品牌色统一
- **用户心理**：这是刚需，AI 要像平台顾问一样给建议
- **Alpha Moment**："它居然知道淘宝主图要 800×800 白底！"

#### Layer 3：审美层 —— "升华它"
- 丁达尔光效、电影级调色、动态景深、插画风格转换
- **用户心理**：用户不知道自己需要，但看到预览后会被惊艳
- **Alpha Moment**："我那张普通照片竟然能变得这么高级！"

```
Layer 3（审美）  ←── Alpha Moment 主要来源
Layer 2（商业）  ←── 付费转化主要驱动
Layer 1（物理）  ←── 留存和信任的基础
```

### 3.3 意图生成的技术架构

**核心原则：AI 负责"看懂"，规则库负责"审美决策"，生成模型负责"精准执行"。**

纯靠模型推理 editableActions 不可靠（会跑偏、会平庸）。可靠的方案是三步过滤法：

```
任意图片
  ↓
Step 1: LLM 语义定标（Semantic Tagging）
  模型不直接想 Action，先打标签
  输出：{ "category": "food", "tags": ["egg_yolk", "meat_patty", "cluttered_bg"] }
  ↓
Step 2: 规则引擎匹配（Expert Mapping）
  根据 category + tags 从专家库提取候选 Actions
  food + egg_yolk → ['enhance_yolk_flow', 'add_steam', 'warm_lighting']
  food + cluttered_bg → ['clean_background', 'replace_with_marble']
  ↓
Step 3: LLM 最终精选 + 个性化描述（Contextual Refinement）
  从候选池中选出最适合这张图的 3-5 个
  输出：{ "action": "enhance_yolk_flow", "description": "让蛋黄看起来更饱满顺滑" }
```

#### 行业专家 Prompt 路由

同一张图在不同业务线下，意图完全不同：

| 业务线 | 同一张人物照片的 editableActions |
|--------|-------------------------------|
| **跨境电商** | 换白底、添加多国语言吊牌、适配 Amazon 规范 |
| **小红书达人** | 日系胶片滤镜、添加手写文字、氛围感光效 |
| **企业 HR** | 统一品牌色、正式背景、证件照裁切 |
| **婚庆摄影** | 电影调色、去背景路人、添加光斑 |

## 四、创意飞轮：数据驱动的视觉迭代

### 4.1 飞轮模型

当图像产生的曝光、点击等数据能够回流时，系统从"工具"进化为"引擎"：

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│              创意飞轮 (Creative Flywheel)             │
│                                                     │
│     ┌──────────────┐                                │
│     │  AI 生成图片   │ ←───── editableActions         │
│     │  (创意执行)    │        (AI 提供创意方向)         │
│     └──────┬───────┘              ↑                  │
│            ↓                      │                  │
│     ┌──────────────┐       ┌──────┴───────┐         │
│     │  投放分发     │       │  创意优化      │         │
│     │  (电商/社媒)  │       │  (LLM 分析    │         │
│     └──────┬───────┘       │   什么有效)    │         │
│            ↓               └──────┬───────┘         │
│     ┌──────────────┐              ↑                  │
│     │  数据回流     │ ────────────┘                  │
│     │  (曝光/点击   │                                │
│     │   /转化/GMV)  │                                │
│     └──────────────┘                                │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 4.2 飞轮运转示例

```
第 1 天：用户上传白底产品图
  → AI 建议 5 种背景风格 → 用户选了"原木极简风" → 发布到店铺

第 7 天：数据回流
  → "原木极简风" CTR 4.2%
  → 但同品类竞品使用"奶油渐变"风格，CTR 达到 6.8%

第 8 天：AI 主动推送
  → "您的产品主图 CTR 低于品类均值 38%。
     建议尝试'奶油渐变'背景，该风格在同类产品中点击率高出 62%。
     需要我生成一版吗？"

第 15 天：积累了 1000 张图的行业数据
  → AI 已自主学到：
     · 母婴品类 → 暖色背景 CTR 更高
     · 3C 数码 → 冷色金属感 CTR 更高
     · 食品类 → 俯拍 > 侧拍
     · 周五发布的图 → 互动率整体 +15%
  → editableActions 的排序自动按转化率加权

第 30 天：系统开始"带预判"地生成
  → 用户上传新产品图时，AI 直接说：
     "基于同品类数据，推荐使用暖色调 + 45° 俯拍 + 左侧打光，
      预估 CTR 可达 7.2%。已为您生成 3 个方案，请选择。"
```

### 4.3 数据回流的技术路径

```
Phase 1（无数据期）：
  editableActions 排序依据 = 行业通用规则 + LLM 审美判断
  
Phase 2（冷启动期）：
  接入第三方数据（淘宝生意经、小红书创作者中心的公开指标）
  进行品类级别的粗粒度优化
  
Phase 3（数据积累期）：
  用户授权回传自己图片的曝光/点击数据
  建立「视觉特征 → 转化率」的关联模型
  
Phase 4（飞轮成熟期）：
  系统能基于历史数据预判每个 editableAction 的 ROI
  按预估 ROI 排序推荐
  最终实现：AI 生成的图 > 人工设计的图（在转化率维度）
```

## 五、图片→JSON：语义理解的技术基座

### 5.1 三级管线架构

```
Level 1: 语义理解层 (LLM Vision)
  任意图片 → 语义 JSON（元素列表 + bbox + editableActions）
  模型：Gemini 2.5 Pro / Flash
  耗时：3-5s | 成本：~$0.01

Level 2: 精确分割层 (SAM)
  JSON 的 bbox → 像素级精确 Mask
  模型：SAM 2 (Meta 开源)
  耗时：~50ms | 成本：极低

Level 3: 精准编辑层 (Inpainting)
  原图 + Mask + 修改指令 → 只修改目标区域
  模型：FLUX Fill / Imagen 3
  耗时：10-20s | 成本：~$0.03-0.05
```

### 5.2 统一 JSON Schema

```json
{
  "canvas": {
    "width": "number",
    "height": "number",
    "category": "string"
  },
  "elements": [
    {
      "id": "string",
      "type": "text | product | character | environment | ...",
      "role": "subject | background | structure | sceneProp | decoration | watermark",
      "content": "string?",
      "description": "string",
      "bbox": { "x": 0, "y": 0, "w": 0, "h": 0 },
      "style": {},
      "subParts": [],
      "zIndex": 0
    }
  ],
  "editableActions": [
    {
      "action": "string",
      "targetId": "string",
      "description": "string",
      "layer": "physical | commercial | aesthetic",
      "estimatedImpact": "string?"
    }
  ]
}
```

### 5.3 Prompt 路由：分类器 + 专家 Prompt 池

```
任意图片
  ↓
分类器 Prompt（Flash，<1s，~$0.001）
  → category: "food_photo"
  ↓
路由到 expert_prompts/food_photo.md
  → 针对美食图的专用分析 Prompt
  → 输出精准 JSON + 高质量 editableActions
```

专家 Prompt 池覆盖：
- `ecommerce_product.md` — 电商产品图
- `poster_design.md` — 海报/Banner
- `food_photo.md` — 美食摄影
- `portrait.md` — 人像/写真
- `art_photography.md` — 艺术摄影
- `fashion_photography.md` — 时装人像
- `real_estate.md` — 房产/空间
- `illustration.md` — 插画/手绘
- `screenshot_ui.md` — UI 截图
- `document.md` — 证件/文档

### 5.4 三模型对比实验结论

同一张图片，Gemini Flash / Gemini Pro / Claude Opus 对比：

| 维度 | Flash | Pro | Opus |
|------|-------|-----|------|
| 格式一致性 | ✅ | ✅ | ✅ |
| 元素完整度 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| subParts 粒度 | 2 个 | 4 个 | 4 个 |
| editableActions 深度 | 4 个基础 | 4 个有创意 | 6 个含细粒度 |
| 构图/叙事分析 | 无 | 无 | 有 |
| bbox 精度 | ±50px | ±50px | ±50px |

**结论**：
- **格式约束（Schema）可以抹平模型差异**
- **内容深度随模型能力递增**，但强 Prompt 可补齐 80-90%
- **生产策略**：首次深度解析用强模型，后续交互用 Flash

## 六、竞品对比与差异化

```
               图片理解        创意建议        数据闭环
               ──────────      ──────────     ──────────
美图设计室      有（生成时记录）  无             无
Canva          有（模板元数据）  有（模板推荐）  无
Adobe Firefly  有（Sensei）     有（基础）     无
               
WebPoster      ✅ LLM 语义解析  ✅ 三层意图模型  ✅ 创意飞轮
(目标)         (适用于任意图片)  (物理/商业/审美) (数据驱动迭代)
```

**WebPoster 的护城河**：
1. **万图可解**：不限于自己生成的图，任何上传的图片都能做语义理解
2. **懂业务线**：同一张图在不同行业下给出不同的策略建议
3. **数据飞轮**：越用越准，竞品无法复制的"数据+创意"壁垒

## 八、总结

```
一句话总结：

当人人都能生成图片时，
真正稀缺的是「知道该生成什么」和「为什么生成这个」。

WebPoster 的使命是成为那个
比用户更懂市场、更懂审美、更懂数据的 AI 视觉策略师。

editableActions 是这一切的起点——
不是告诉用户「你可以改什么」，
而是告诉用户「你应该改什么，以及为什么」。
```
